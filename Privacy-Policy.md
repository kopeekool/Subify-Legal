# Privacy Policy

*Last Updated: May 28, 2026*

## Subify Privacy Policy

Thank you for using **Subify** ("the Bot", "we", "our", "us").

This Privacy Policy explains what information Subify collects, how it is used, stored, and protected.

By using Subify, you agree to this Privacy Policy.

---

## 1. Information We Collect

Subify may collect and store limited Discord-related data required for bot functionality.

### User Data

We may collect:

* Discord User IDs
* Discord Server IDs
* Discord Channel IDs
* Verification status records
* Warning / moderation records
* Appeals and notes
* Cooldowns and verification attempt history

### Uploaded Verification Content

When using `/verify`, users may upload screenshots for subscriber verification.

Submitted screenshots may be:

* Processed using OCR (Optical Character Recognition)
* Temporarily analyzed for verification validation
* Logged for moderation or audit purposes

Subify does **not** intentionally collect passwords, payment details, or private credentials.

---

## 2. How We Use Data

Collected data is used solely to:

* Provide verification services
* Validate YouTube subscription screenshots
* Restore verification roles
* Prevent abuse, spam, or fraud
* Support moderation actions
* Generate analytics and dashboard statistics
* Maintain bot functionality and reliability

---

## 3. Data Storage

Subify stores data locally using JSON files.

Example files include:

* `servers.json`
* `subscribers.json`
* `logs.json`
* `warnings.json`
* `appeals.json`
* `stats.json`

No external database is required by default.

Self-hosted bot owners are responsible for securing their own hosting environment.

---

## 4. Data Sharing

We do **not** sell, rent, or trade user data.

Data may only be shared:

* When required by law
* To comply with legal requests
* To investigate abuse, fraud, or security incidents

---

## 5. Data Retention

Data is retained only as long as necessary for bot operations.

Server administrators may reset configuration or moderation data using available bot commands where supported.

Self-hosted deployments control their own stored data.

---

## 6. Security

We take reasonable measures to protect stored information.

However, no system is completely secure, and we cannot guarantee absolute security.

Users and server owners are responsible for securing their Discord accounts and hosting environments.

Use of these services is governed by their respective policies.
